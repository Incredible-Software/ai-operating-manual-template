# Start here

Welcome to your **AI Operating Manual template**. This is the empty starting point. By the time you finish reading this page and doing the three steps below, you'll have a working Operating Manual for your own business.

Takes 2 minutes to read. About 15 minutes to fill in.

## What an Operating Manual is

A set of plain-English markdown files that describe your business — who you are, how you sound, how you work, what your codes mean, and the repeated tasks you do. Load these files into any AI tool (Claude, Copilot, ChatGPT) and the AI's output stops sounding generic and starts sounding like your team wrote it.

## Do these three things, in order

**1. Get your own copy of this template.**

Three options, from easiest to most technical:

- Click the green "Code" button → "Download ZIP." No git knowledge needed.
- Click "Use this template" → "Create a new repository" if you have GitHub. One click.
- Or `git clone` from the command line.

**2. Run the builder skill.**

Open Claude (Pro or Team). Start a new chat. Click the paperclip icon and attach `skills/build-operating-manual/SKILL.md`. Type:

> *"Run the build-operating-manual skill. Interview me about my business."*

Claude will ask one question at a time. Answer in plain language. It drafts COMPANY.md, BRAND.md, PROCESSES.md, GLOSSARY.md, and a starter SKILL.md as it goes.

**3. Save the drafts into your template.**

Replace the placeholder content in each file with the drafts Claude just produced. Keep the file structure the same — the structure is what makes the manual portable across tools.

That's it. You now have an Operating Manual.

## Want to see what "finished" looks like?

Browse the [Anka Civil reference repo](https://github.com/Incredible-Software/ai-brain-demo) — a worked example for a fictional 40-person NZ engineering consultancy. It's the same file structure, fully filled in. Look at it for tone, length, and texture. Don't fork it. Use this template instead.
